这是一个用来辅助交易的微型量化交易系统，我们的名字叫做星火量化，星星之火，点燃你的交易人生。

## 架构设计：

- 控制台： 用来进行图表、信号的展示以及用户交互
- 主程序： 
    - web api: 提供前端交互用的 API 接口
    - 行情抓取： 根据配置，抓取多个交易市场的行情数据
    - 实时监测： 根据行情发展的需要，由策略分析模块提出对某个交易标的的实时监测需求，以便在其价格达到某个位置的时候实时触发
    - 公共数据： 根据行情抓取的数据，进行公共的计算和数据的维护，例如计算 ema 数据
    - 策略分析： 接收行情抓取的K 线信息，进行各自的策略分析，例如箱体监测、趋势监测、阻力支撑位监测以及突破监测、量价异常的监测
    - 消息通知： 通过将交易信号发送到飞书的方式，产生提醒，我会提供飞书的群聊信息和发送的认证信息。
    - 交易跟踪： 通过进行本地的模拟交易，对交易信号的质量和交易策略的质量进行评估和统计。
- 数据库： 使用 postgres 作为数据库进行保存，版本是 postgres:18.0

## 详细设计

### 控制台

1. 简单的用户登录
2. 根据不同数据来源，不同策略，查看交易信号，以及对应的基本数据，例如箱体、趋势，阻力位置和支撑位置。
3. 查看交易模拟的统计情况。
### 行情抓取

市场划分：
1. bybit：bybit 作为币圈 API 提供较为完善的交易所，并且提供期权的合约交易。我们采用bybit 合约作为行情获取的来源
2. A 股： 抓取 A 股的行情数据
3. 美股： 提供美股的行情数据

市场周期：
1. bybit: 币圈交易比较频繁，可以提供 15m 、 1h 这两个级别的行情数据
2. A 股和美股提供 1 天的行情分析数据

市场范围：
1. bybit ： 根据热度取 100-200 个币对，范围包括最近 1 个月内上过涨幅榜和跌幅榜前 50 名的；具体方法可以是每天更新 100 名 币对，数据库记录币对上榜的最后时间，超过 1 个月未有变化的币对，不再跟踪。并且每天排序仅取前 200 名。
2. A 股市场： 根据热度同样取 200 个标的，热度判定取近一个月上过涨停的股票，同样记录最后一次涨停的时间；
3. 美股市场： 根据热度同样取 200 个标的，热度判定取周线周期级别处于上行趋势的标的；

行情抓取根据市场、周期进行组合排列，允许并行多个行情抓取器；并且在配置文件中，可以开启和关闭；

建议通过工厂模式管理行情抓取器。

为了避免对数据来源的频繁抓取，建议将行情数据存储到本地数据中。
行情模块提供 K 线查询的接口，优先查询本地，当需要的 K 线范围本地存在时，则查询数据来源，并将数据继续存储到本地中。
### 实时监测

由策略触发的实时监测器，每一个标的（币或者股票）只有一个唯一的实时监测器。
策略每触发一次实时监测器，则检测器的订阅数+1，多个策略需要同一个实时检测器的啥时候，只增加订阅数；
实时检测器可以同时监测多个目标价格，每个目标价格归属一个订阅，当触达目标时，将事件发送给对应的策略分析器；

当策略分析器不再需要实时监测的时候，需要撤销订阅。当订阅为 0 的时候，实时监测器需要被关闭。

建议通过工厂模式，管理实时监测器。

### 公共数据

每一个标的都由公共模块维护它对应的 ema 数据。将 ema 数据保存到行情数据存储的对应数据表中，只需要增加一个 ema 的字段。
由于 ema 需要历史 K 线数据，则由公共模块调用行情模块的接口，获取需要的历史 K 线数据，并存储到本地。

ema 包括短期、中期、长期，默认取 30 、 60 、 90 。
### 策略分析

由工厂模式，管理多个策略分析器。策略分析器是量化逻辑的主要核心模块，通过 channel 监听行情抓取和实时监测模块提供的最新 K 线数据，然后进行策略分析。
策略分析模块如果需要历史 K 线数据，则调用行情模块获取。

#### 1. 箱体突破策略

系统监控行情数据，抓取行情中产生的交易箱体，也叫交易区间。具体如何抓取交易区间，需要你帮我设计算法。
当区间形成后，就保存到箱体数据库表中，箱体分为活跃状态和关闭状态。当价格突破了箱体上下边界之后，就关闭箱体，并触发箱体突破信号。
箱体突破信号同样要保存到数据库中，并且作为交易信号的一种进行保存，统一存储到交易信号中。

为了保证实时性，活跃箱体产生后，触发订阅实时行情数据，当箱体结束后，取消对实时行情的订阅。

为了处理服务重启的逻辑，需要对数据库中已经存在的活跃箱体进行判断，箱体是否仍处于活跃状态，是否需要关闭。

#### 2. 趋势逻辑

受限判断趋势形成，分为短期、中期、长期均线。根据短期 > 中期 > 长期，判断多头趋势，反之则判断空头趋势，以及震荡趋势。

趋势判断出来后，要保存到数据库中，标记为活跃趋势，直到趋势状态不存在，则标记趋势结束。

趋势回测信号是我们要抓取的交易信号，趋势回撤到短期、中期、长期均可产生交易信号，只是强度不同。

#### 3. 阻力和支撑关键位

分析出行情目前关键的阻力位和支撑位，例如前高、此前高、或者前低、次前低这样的关键位置，每当价格在上升图中突破了关键位，或者在下跌过程中突破了关键位，则触发交易信号。

一旦标示出阻力和支撑，则在数据库中进行记录，仅记录前高、此前高、或者前低、次前低四个关键位置即可。

#### 4. 量价异常

一旦出现了价格超过了最近若干根 K 线的平均波动幅度的 2 倍以上，或者成交量突破了 2 倍以上，均触发交易信号，产生包括量价齐升之类的信号，或者突然下跌这样的信号，我们通过抓取这样
的信号，提供交易机会。

#### 交易跟踪

1. 每个交易信号，产生一个交易跟踪；每个活跃的交易信号，同样向实时行情获取器产生一个订阅，直到交易平仓后，取消订阅；
2. 每一个交易信号，根据交易类型，合理设置止盈位置和止损位置。具体如何设置，需要你帮我完善。
3. 控制台需要展示交易跟踪的数据分析情况，帮我设计数据分析和展示的图表。

##### bybit 模拟交易
后期针对币的类型，可以开放 bybit 模拟账户交易，自动根据交易信号进行对应仓位的下单。
