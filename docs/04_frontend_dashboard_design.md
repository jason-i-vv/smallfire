### 1.3 布局特点

| 特性 | 说明 |
|------|------|
| 侧边栏宽度 | 默认 240px，折叠后 64px |
| 可折叠菜单 | 支持鼠标点击折叠/展开侧边栏 |
| 菜单分组 | 按功能模块分组（信号、市场、交易、设置） |
| 顶部栏 | 显示当前页面标题、系统状态、用户信息 |
| 响应式 | 小屏幕下自动使用折叠模式 |

## 6. 技术栈选型

## 7. API 接口设计

### 7.1 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/signals | GET | 获取信号列表 |
| /api/signals/:id | GET | 获取信号详情 |
| /api/trades | GET | 获取交易记录 |
| /api/trades/stats | GET | 获取统计数据 |
| /api/positions | GET | 获取当前持仓 |
| /api/positions/:id/close | POST | 平仓 |
| /api/klines | GET | 获取K线数据 |
| /api/equity | GET | 获取权益曲线数据 |

### 7.2 WebSocket 接口

| 事件 | 方向 | 说明 |
|------|------|------|
| kline:update | Server→Client | K线实时更新 |
| signal:new | Server→Client | 新信号通知 |
| position:update | Server→Client | 持仓更新 |
| trade:closed | Server→Client | 交易平仓通知 |

## 8. 响应式设计

### 8.1 断点设置

| 断点 | 宽度 | 布局 |

| Desktop | 1024px - 1440px | 标准三列布局 |
