# 趋势回撤策略修复与回测图表优化

## 日期：2026-04-14（第二轮修复）

## 第二轮修复：策略核心逻辑重写

### 新发现的问题

1. **依赖EMA排列检测趋势导致漏检**：策略要求EMA三线严格排列（30>60>90 或 30<60<90）且间距足够才认定趋势。但暴跌时EMA来不及拉开距离，导致趋势被误判为 sideways。例如 2.23 暴跌时 EMA30=1968.23, EMA60=1968.20, EMA90=1969.14（三条粘合），被判定为横盘，无法触发回撤信号。

2. **wasFarFromEMA回看窗口过窄**：原逻辑检查15-30根K线前是否远离EMA，但暴跌发生在12根K线前，恰好在窗口之外。

3. **熊市回撤用HIGH而非LOW判断远离EMA**：检查之前的HIGH是否低于EMA，但暴跌场景应该检查LOW。

4. **无冷却机制导致连续触发**：同一标的每根K线都能触发信号，1.23日曾出现连续5根K线各触发一个信号。

### 解决方案：彻底重写策略

**新策略核心逻辑**：不再依赖EMA排列判断趋势，而是对每条EMA独立检测回撤。

```
对 EMA90 / EMA60 / EMA30 分别检查（优先长周期）：
1. EMA必须有方向性：10根K线内变化>0.3%（横盘不触发）
2. 当前K线的低/高点触及EMA（±0.8%范围内）
3. 收盘价确认EMA为支撑/阻力
4. 5-25根K线前，价格曾远离EMA至少2%
5. 冷却：8根K线内不重复触发
```

- 牛市回撤：EMA上升 + 低点触及EMA + 收盘在EMA上方 + 之前高点远高于EMA
- 熊市回撤：EMA下降 + 高点触及EMA + 收盘在EMA下方 + 之前低点远低于EMA

|------|--------|----------|----------|

## 第一轮修复：2026-04-13

### 问题背景

1. 趋势策略依赖 `TrendRepo.GetActive()` 查实时数据库，回测无法工作
2. 回撤检测逻辑反了（要求价格跌破EMA而不是接近EMA）
3. 回测图表没有EMA均线和趋势回撤标记
4. 策略命名"趋势跟踪"改为"趋势回撤"
