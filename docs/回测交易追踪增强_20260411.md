# 回测交易追踪模块增强

## 变更日期
2026-04-11

## 需求背景
回测功能已有基础交易模拟（`enable_trade=true`），能生成交易记录和统计指标，但存在以下不足：
1. K线图上无法直观看到交易买卖点和持仓区间
2. 交易记录表格信息不够丰富（缺少收益率、累计盈亏等）
3. 不支持移动止损策略
4. 缺少交易分析面板

## 变更内容

### 1. K线图交易可视化（核心交付）
- **BacktestChart.vue**: 新增 `trades` prop，接收交易记录
- 在K线图上叠加交易标记：入场▲/▼（绿色=做多，红色=做空），出场标记显示盈亏金额
- 利用 overlay canvas 绘制持仓区间半透明色带：绿色=盈利，红色=亏损
- 添加交易图例说明
- 信号标记和交易标记合并去重，优先显示交易标记

### 2. 交易记录表格增强
- 新增"收益率"列和"累计盈亏"列
- 行级着色：盈利行绿色背景，亏损行红色背景
- 新增可折叠"交易分析"面板，包含：
  - 多头胜率 / 空头胜率
  - 平均持仓时间
  - 最佳/最差交易
  - 平均盈利/平均亏损
  - 最大连胜/连亏

### 3. 后端移动止损支持
- **BacktestRequest**: 新增 `trailing_stop_pct`（移动止损距离）和 `trailing_activate_pct`（激活距离）
- **BacktestTrade**: 新增 `trailing_stop_activated` 和 `max_favorable_excursion` 字段
- **backtest_service.go**: `runBacktestLoop` 增加移动止损状态机
  - 出场优先级：移动止损 > 固定止损 > 止盈
  - 参考 `internal/service/trading/trailing_stop.go` 的实现逻辑

### 4. 前端移动止损控件
- 回测参数表单增加"移动止损"开关
- 开启后显示"止损距离"和"激活距离"滑块
- 参数传递给后端

## 涉及文件
| 文件 | 改动类型 |
|------|---------|
| `internal/models/backtest.go` | 修改（移动止损字段） |
| `internal/service/backtest/backtest_service.go` | 修改（移动止损逻辑、MFE） |
| `starfire-frontend/src/components/charts/BacktestChart.vue` | 修改（交易可视化） |
| `starfire-frontend/src/views/backtest/Backtest.vue` | 修改（表格增强、分析面板、表单） |

## 使用方式
1. 在回测页面开启"执行交易"开关
2. 可选开启"移动止损"设置止损距离和激活距离
3. 执行回测后，在信号列表的图表模式中查看交易标记和持仓色带
4. 点击"展开分析"查看详细交易分析
