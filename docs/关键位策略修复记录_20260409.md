# 关键位策略修复记录

**日期**: 2026-04-09
**问题来源**: 回测 `backtest_1000000CHEEMSUSDT_1h_key_level_20260409_103155.json` 数据分析

## 问题概述

1. 同一K线同时产生 `resistance_break`（做多）和 `support_break`（做空）矛盾信号
2. 信号描述不准确：支撑位应描述为"跌破"而非"突破"
3. 回测与实盘策略参数、触及判定、强度计算均不一致
4. 触及判定逻辑不合理：大实体K线不会被计为触及
5. 关键位列表无限增长，没有过期/清理机制
6. 突破选位逻辑不合理：取最高/最低价位而非最接近当前价的
7. 回测与实盘触及计数范围不一致
8. 波峰波谷检测没有收盘确认，长影线噪音被误认为波峰波谷

## 修复内容

### 1. 同一K线矛盾信号互斥
- **文件**: `internal/service/backtest/backtest_service.go`, `internal/service/strategy/key_level_strategy.go`
- **方案**: 新增 `resolveConflictingSignals()` 函数，当同一K线同时产生 long 和 short 信号时，选择突破距离更大的一方；距离相近（差异 < 30%）时取触及次数更高的一方

### 2. 触及判定逻辑修正
- **文件**: `internal/service/backtest/backtest_service.go` `countTouchSingle()`, `internal/service/strategy/key_level_strategy.go` `updateTouchCounts()`
- **修改前**: 阻力位 `ClosePrice < levelPrice + 0.3%`，支撑位 `ClosePrice > levelPrice - 0.3%`
- **修改后**: 阻力位 `ClosePrice <= levelPrice`，支撑位 `ClosePrice >= levelPrice`
- **含义**: 触及 = 影线触达但实体未收过价位，避免大实体K线被错误排除

### 3. 强度计算统一
- **文件**: `internal/service/backtest/backtest_service.go` `calculateStrength()`
- **修改前**: 基于附近同类型价位数量（1-3）
- **修改后**: 基于触及次数（1-5），与实盘策略一致

### 4. 回测与实盘参数统一
- **文件**: `internal/config/config.go`, `config/config.yml`, `internal/service/backtest/backtest_service.go`, `internal/service/strategy/key_level_strategy.go`, `cmd/server/main.go`
- **配置变更**:
  - `lookback_klines`: 20 → 50（与回测保持一致）
  - `level_distance`: 0.5% → 0.2%（与回测保持一致）
  - 新增 `min_breakout_age: 5`（价位最小成熟期）
- **回测策略**: `newKeyLevelStrategyAnalyzer()` 改为从配置读取参数，不再硬编码
- **实盘策略**: 增加 `minBreakoutAge` 检查，触及次数未达标的价位不产生突破信号

### 5. 信号描述文本修正
- **文件**: `internal/service/strategy/key_level_strategy.go`, `internal/service/backtest/backtest_service.go`
- 阻力位 → "突破阻力位，触及N次，..."
- 支撑位 → "跌破支撑位，触及N次，..."

### 6. signal_handler 补充遗漏类型
- **文件**: `internal/handler/signal_handler.go`
- `signalTypes` 列表补充 `support_break`

### 7. 关键位过期清理机制
- **文件**: `internal/service/backtest/backtest_service.go`, `internal/repository/key_level_repo.go`, `internal/service/strategy/key_level_strategy.go`
- **回测**: `LevelInfo` 新增 `lastTouchIter` 字段；`Analyze()` 开头调用 `filterExpiredLevels()` 清理超过 `lookbackKlines*2` 根K线未触及且未突破的价位
- **实盘**: `GetActive()` 查询增加 `valid_until > NOW()` 过滤；创建价位时设置 `ValidUntil = now + lookbackKlines*2*periodDuration`

### 8. 突破选位逻辑优化
- **文件**: `internal/service/backtest/backtest_service.go`, `internal/service/strategy/key_level_strategy.go`
- **修改前**: 阻力位取最高的（最强阻力），支撑位取最低的（最强支撑）
- **修改后**: 阻力位取最接近当前价的（最低的被突破阻力），支撑位取最接近当前价的（最高的被跌破支撑）
- **理由**: 最先被突破的价位信号更早、止损更近、风险收益比更好

### 9. 触及计数范围统一
- **文件**: `internal/service/strategy/key_level_strategy.go` `updateTouchCounts()`
- **修改前**: 扫描最近10根K线取最大值
- **修改后**: 只检查最新1根K线，累计计数（与回测 `countTouchSingle` 一致）

### 10. 波峰波谷收盘确认
- **文件**: `internal/service/backtest/backtest_service.go` `detectKeyLevelSwingPoints()`, `internal/service/strategy/key_level_strategy.go` `detectSwingPoints()`
- **新增条件**: 波峰要求 `ClosePrice > (HighPrice + LowPrice) / 2`，波谷要求 `ClosePrice < (HighPrice + LowPrice) / 2`
- **含义**: 收盘在K线中位以上才认定为有效波峰，过滤长上/下影线噪音


## 对抗评审修复（第二轮）

### 11. Create() SQL 参数数量不匹配（P0）
- **文件**: `internal/repository/key_level_repo.go`
- **问题**: SQL 有 8 个占位符（$1-$8），但 `QueryRow` 只传了 7 个参数，缺少 `level.ValidUntil`
- **修复**: 添加 `level.ValidUntil` 为第 8 个参数

### 12. countTouchSingle 未更新 lastTouchIter（P0）
- **文件**: `internal/service/backtest/backtest_service.go` `countTouchSingle()`
- **问题**: 触及时只增加 `TouchCount` 但未更新 `lastTouchIter`，导致 `filterExpiredLevels` 始终使用 `createdAtIter`
- **修复**: 触及时同时设置 `level.lastTouchIter = a.iteration`

### 13. minBreakoutAge 语义统一（P1）
- **文件**: `internal/service/backtest/backtest_service.go`
- **问题**: 回测使用 `klinesAway`（迭代差）判定成熟度，实盘使用 `KlinesCount`（触及次数）判定
- **修复**: 回测改为使用 `level.TouchCount < a.minBreakoutAge`，与实盘一致

### 14. SignalData key 名称统一（P1）
- **文件**: `internal/service/backtest/backtest_service.go`
- **问题**: 回测使用 `touch_count`，实盘使用 `klines_count`
- **修复**: 统一为 `klines_count`；同时统一描述格式为 "突破阻力位，触及N次，距突破X%"

### 15. FindActive/GetBySymbol 补充 valid_until 过滤（P2）
- **文件**: `internal/repository/key_level_repo.go`
- **问题**: `GetActive` 有 `valid_until` 过滤但 `FindActive` 和 `GetBySymbol` 没有
- **修复**: 两个查询都添加 `AND (valid_until IS NULL OR valid_until > NOW())` 条件

### 16. 实盘 detectSwingPoints 添加收盘确认（P4）
- **文件**: `internal/service/strategy/key_level_strategy.go` `detectSwingPoints()`
- **问题**: 回测已添加 `ClosePrice > midPrice` 收盘确认条件，实盘遗漏
- **修复**: 波峰添加 `ClosePrice > midPrice`，波谷添加 `ClosePrice < midPrice`

### 17. 新价位创建初始化 lastTouchIter
- **文件**: `internal/service/backtest/backtest_service.go` `Analyze()`
- **问题**: 新创建的 `LevelInfo` 未初始化 `lastTouchIter`（默认 0），虽然不影响正确性但语义不清晰
- **修复**: 创建时设置 `lastTouchIter: a.iteration`
