# 策略优化与 Bug 修复

## 变更日期
2026-04-12

## 需求背景
基于对 6 个策略（box、trend、key_level、volume_price、wick、candlestick）在 5 个标的上（BTCUSDT、ETHUSDT、SOLUSDT、DOGEUSDT、DUSKUSDT）的回测评估，发现以下问题：
1. candlestick 策略信号状态错误导致无法执行交易
2. 策略工厂单例模式导致回测结果不可复现
3. nil slice JSON 序列化为 null 导致前端解析失败
4. 盈亏比和最大回撤计算公式错误
5. key_level 策略信号噪声过大
6. wick 策略只产生做空信号，做多信号为 0
7. trend/box 策略效果差

## 变更内容

### P0 Bug 修复

#### 1. candlestick 策略信号状态修复
- **文件**: `internal/service/strategy/candlestick_strategy.go`
- **问题**: 信号状态硬编码为 `SignalStatusActive`，但回测交易执行逻辑只处理 `SignalStatusPending`
- **修复**: 改为 `SignalStatusPending`

#### 2. 回测策略工厂单例问题修复
- **文件**: `internal/service/strategy/factory.go`
- **问题**: `GetStrategy` 返回单例实例，策略内部有 cooldown 状态，第二次回测复用已冷却的单例导致 0 信号
- **修复**: 新增 `NewStrategy` 方法创建全新策略实例，回测使用 `NewStrategy` 代替 `GetStrategy`

#### 3. nil slice JSON 序列化修复
- **文件**: `internal/service/backtest/backtest_service.go`
- **问题**: Go nil slice 序列化为 JSON `null`，前端期望 `[]`
- **修复**: 回测响应构建前初始化 nil slice 为空 slice

#### 4. 盈亏比和最大回撤计算修复
- **文件**: `internal/service/backtest/backtest_service.go`
- **问题**:
  - 盈亏比使用 `AvgWin/AvgLoss`，当全部盈利时除以 0 报错
  - 最大回撤只在交易结算时计算，不包含持仓期间浮盈亏
- **修复**:
  - 盈亏比改为 `totalWin/totalLoss`
  - 在 `runBacktestLoop` 中逐 K 线计算权益曲线和最大回撤（含浮盈亏）

### P1: key_level 策略信号降噪
- **文件**: `internal/service/backtest/backtest_service.go`
- **问题**: BTCUSDT 1 个月产生 130 个信号，其中 strength 1 占 65%，大部分是噪声
- **修复**:
  1. strength < 3 的信号在回测中过滤（信号列表和交易执行均使用过滤后的信号）
  2. 突破信号冷却机制：每次突破后 10 根 K 线内不再产生新突破信号（冷却检查移到信号生成之前）
  3. `minBreakoutAge` 默认值从 5 调整为 10
  4. 波峰波谷识别回溯从 3 调整为 5
  5. `calculateStrength` 增强：综合考量 `TouchCount` 和 `klinesAway`
- **效果**: BTCUSDT 信号数从 130 降至 29（1 个月），strength 分布从 {1:84, 2:7, 3:10, 4:5, 5:24} 变为 {3:6, 4:9, 5:14}

### P2: wick 策略做多信号修复
- **文件**: `internal/service/strategy/wick_strategy.go`
- **问题**: `require_trend: true` 配置下，下引线反转需要 bearish 趋势，牛市中几乎不触发。195 个信号全部是做空
- **修复**:
  1. 移除趋势硬性过滤（趋势不匹配时不再返回 nil）
  2. `calculateStrength` 中增加趋势一致性逻辑：趋势一致时加成，不一致时降低强度
- **效果**: BTCUSDT 1 个月 84 个信号，42 做多 42 做空，信号完全对称

### P3: trend/box 策略评估结论
- **trend**: 1 个月 0 信号，几乎失效。只保留 `trend_retracement` 且条件过严，需要重新设计信号逻辑
- **box**: 已默认禁用（`enabled: false`），胜率 28%，当前市场效果差
- **处理**: 保持代码不变，后续有需要时重新设计

3. 验证 candlestick 策略能产生交易
4. 验证同一回测参数多次执行结果一致
5. 验证 key_level 信号 strength >= 3
6. 验证 wick 策略同时有做多和做空信号
7. 验证盈亏比不再出现除以 0 的情况
8. 验证最大回撤百分比不为 0（有交易时）
