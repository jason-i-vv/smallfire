# 回测引线策略图表视图优化

**日期**: 2026-04-06

## 需求背景

回测页面的引线策略（wick）信号展示之前只有列表模式（表格），用户需要逐个点击"图表"按钮跳转到 K 线图页面查看单个信号，无法统一查看回测效果。

## 变更内容

### 1. 新建 BacktestChart 组件

**文件**: `starfire-frontend/src/components/charts/BacktestChart.vue`

内嵌式 K 线图表组件，核心功能：
- 通过 `klineApi.list()` 获取回测时间范围内的 K 线数据
- 使用 lightweight-charts 创建蜡烛图 + 成交量图
- 通过 `setMarkers()` 在所有信号位置添加箭头标记，4种信号类型使用不同颜色：
  - 上引线反转 (upper_wick_reversal): 红色, arrowDown
  - 下引线反转 (lower_wick_reversal): 绿色, arrowUp
  - 假突破上引 (fake_breakout_upper): 橙色, arrowDown
  - 假突破下引 (fake_breakout_lower): 黄色, arrowUp
- 使用 overlay canvas 在每个引线信号位置绘制颜色竖线标记
- 添加信号图例栏，显示各信号类型的数量

### 2. 修改 Backtest 页面

**文件**: `starfire-frontend/src/views/backtest/Backtest.vue`

- 信号列表卡片头部添加「图表/列表」切换按钮组（仅引线策略显示）
- 引线策略默认展示图表模式
- 图表模式下渲染 BacktestChart 组件（v-if 条件渲染，切换时销毁/重建）
- 列表模式下保持原有表格功能不变
- 切换策略类型时自动重置视图模式
- 从 sessionStorage 恢复时也正确设置视图模式

## 技术要点

- 无需后端 API 变更，使用现有 `/klines` 接口
- 无需路由变更
- 时间戳处理：回测时间为 UTC+8 的 `YYYY-MM-DD HH:mm:ss` 格式，转换为 UTC 秒级时间戳传给 API
- 信号时间对齐：使用 `alignTimeToPeriod()` 将信号时间对齐到 K 线周期起点
