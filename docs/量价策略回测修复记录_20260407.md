# 量价策略回测修复记录

## 日期：2026-04-07

## 问题概述

量价策略(volume_price_strategy)回测时发现以下问题：
1. 同一K线触发了两个信号（价格异常+成交量异常同时满足时）
2. 冷却机制在回测中完全失效，使用 `time.Now()` 导致回测中几乎只能产生第一个信号
3. 信号遗漏（由问题2导致，冷却误杀）
4. 前端只有引线策略支持图表/列表双视图，量价策略缺少图表展示

## 修复内容

### 1. 后端：冷却机制修复（volume_strategy.go）

**问题：** `lastPriceSignal` 和 `lastVolumeSignal` 使用 `time.Now()` 记录冷却时间，用 `time.Since()` 判断冷却。回测时数百根K线在毫秒内处理完毕，导致第一个信号触发后，后续所有K线都在冷却期内。

**修复：** 改为基于K线时间的冷却机制：
- 结构体字段 `lastPriceSignal` → `lastPriceKlineTime`，`lastVolumeSignal` → `lastVolumeKlineTime`
- 冷却判断：`time.Since(s.lastPriceSignal)` → `latest.OpenTime.Sub(s.lastPriceKlineTime)`
- 触发记录：`s.lastPriceSignal = time.Now()` → `s.lastPriceKlineTime = latest.OpenTime`

### 2. 后端：同一K线单信号（volume_strategy.go）

**问题：** `Analyze()` 对同一根K线分别调用 `checkPriceAnomaly` 和 `checkVolumeAnomaly`，可能产生两个信号。

**修复：** 在 `Analyze()` 返回前，如果收集到多个信号，只保留 Strength 最高的那个。

### 3. 前端：量价策略图表展示

**BacktestChart.vue 修改：**
- `SIGNAL_MARKER_CONFIG` 增加4种量价信号类型的标记配置
- `WICK_OVERLAY_STYLES` 重命名为 `SIGNAL_OVERLAY_STYLES`，增加量价信号overlay样式
- `signalTypeLegend` 增加量价信号图例
- `getSignalTypeName` 增加量价信号名称映射
- `buildOverlaySignals` 扩展支持的信号类型白名单

**Backtest.vue 修改：**
- 新增 `isVolumeStrategy` 和 `supportsChartMode` 计算属性
- 模板中 `isWickStrategy` → `supportsChartMode`（图表/列表切换按钮和图表渲染条件）
- `watch` 和 `runBacktest`、`restoreFromSession` 中扩展视图模式切换逻辑

## 2026-04-11 补充修复：price_surge 涨跌方向区分

**问题：** `checkPriceAnomaly` 只基于波动率触发信号，信号类型固定为 `price_surge`，不区分涨跌方向。导致阴线（空方主导）也显示为"价格飙升"，语义矛盾。

**修复：** 信号类型从 `price_surge` 拆分为 `price_surge_up`（阳线，涨）和 `price_surge_down`（阴线，跌），与 `volume_price_rise` / `volume_price_fall` 保持一致的设计模式。
