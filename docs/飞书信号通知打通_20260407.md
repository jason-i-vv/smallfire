# 飞书信号通知打通

**日期**: 2026-04-07

## 变更背景

之前的信号通知链路是断裂的：
- `runner.go:170` 的 `r.sendNotification(signal)` 被注释掉，带 TODO 标记
- `Runner` struct 没有 `notifier` 字段，即使取消注释也无法发送
- 唯一在运行的通知是定时交易汇总（09:00/15:00/21:00），但因为交易功能已关闭，汇总数据全是零
- `feishu.go` 的 `getSignalTypeName()` 缺少 wick 策略的信号类型映射，会直接显示英文原始值

## 变更内容

### 1. Runner 注入 Notifier

**文件**: `internal/service/strategy/runner.go`
- 新增 `DependencyNotifier` 接口定义
- `Runner` struct 添加 `notifier` 字段
- `NewRunner` 增加 `notifier` 参数
- 信号创建成功后调用 `r.notifier.SendSignal(signal)` 发送飞书通知
- 删除空的 `sendNotification` stub 方法

**文件**: `cmd/server/main.go`
- `strategy.NewRunner()` 调用处传入 `notifyManager`

### 2. 补全信号类型映射

**文件**: `internal/service/notification/feishu.go`
- `getSignalTypeName()` 添加：
  - `upper_wick_reversal` → "上引线反转"
  - `lower_wick_reversal` → "下引线反转"
  - `fake_breakout_upper` → "假突破上引线"
  - `fake_breakout_lower` → "假突破下引线"

### 3. 优化飞书卡片格式（精简版）

**文件**: `internal/service/notification/feishu.go`
- 标题格式：`🟢 BTCUSDT 上引线反转`（标的代码 + 信号类型）
- 内容只保留：方向、强度、信号价格
- 去掉无意义的 `Data` 字段（`symbol_id` 等内部 ID 对用户无意义）

### 4. 关闭交易汇总

**文件**: `config/config.yml`
- `feishu.send_summary` 设为 `false`，避免推送全零数据的交易汇总
