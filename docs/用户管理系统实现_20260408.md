# 用户管理系统实现记录

**日期**: 2026-04-08
**需求**: 为星火量化交易平台添加用户注册、登录、JWT 认证、角色权限控制和账户管理功能

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 用户注册 | 公开接口，用户名 3-32 位字母数字下划线，密码 6-64 位 |
| 用户登录 | 返回 JWT token，统一错误提示防止枚举 |
| JWT 认证 | HMAC-SHA256 签名，24h 过期，每请求校验用户状态 |
| 角色权限 | admin/user 两种角色，RBAC 中间件控制 |
| 修改密码 | 需验证旧密码，修改后需重新登录 |
| 用户管理 | admin 可查看用户列表、禁用/启用用户、重置密码 |

## API 接口

### 公开接口（无需认证）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/auth/register` | 用户注册 |
| POST | `/api/v1/auth/login` | 用户登录 |

### 认证接口（需 Bearer Token）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/auth/me` | 获取当前用户信息 |
| PUT | `/api/v1/auth/password` | 修改密码 |

### 管理员接口（需 admin 角色）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/users` | 用户列表 |
| PUT | `/api/v1/users/:id/status` | 启用/禁用用户 |
| PUT | `/api/v1/users/:id/password` | 重置用户密码 |

### 请求/响应示例

**注册**:
```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"test123456","nickname":"测试用户"}'
```

**登录**:
```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"test123456"}'
```

**带 Token 访问**:
```bash
curl http://localhost:8080/api/v1/signals \
  -H "Authorization: Bearer <token>"
```

## 配置说明

在 `config/config.yml` 中需要配置 JWT 密钥：

```yaml
jwt:
  secret: "your-jwt-secret-key"  # 生产环境建议通过 JWT_SECRET 环境变量设置
  expiration: "24h"
```

服务启动时会检查 JWT 密钥是否配置，未配置则直接 panic 退出。

## 错误码

| 错误码 | 说明 | HTTP 状态码 |
|--------|------|------------|
| 10001 | 参数验证失败 | 400 |
| 10002 | 未认证/认证无效 | 401 |
| 10003 | 认证已过期 | 401 |
| 10004 | 权限不足 | 403 |
| 10008 | 用户已被禁用 | 403 |

## 部署注意事项

1. **前后端需同步部署**：后端部署后所有 API 需要 Token，前端未部署前请求会全部 401
2. **JWT_SECRET 环境变量**：生产环境必须设置强密钥
3. **数据库表**：`users` 表已存在，`username` 字段已有 UNIQUE 约束
