# 关键价位算法优化记录

日期: 2026-04-14

## 背景

关键价位（支撑位/阻力位）识别从 AI 方案切换回算法方案后，存在以下问题：
1. 整数关口检测粒度太细（ETH 2340 只是 10 的倍数，不是真正心理关口）
2. 评分选位逻辑强制每种强度选一个，导致弱位填充
3. Swing point 检测没有显著性过滤，微小波动也产生候选
4. 聚类容忍度固定 0.3%，不适应不同波动率品种
5. 最小触碰次数为 1，噪音价位入选

同时参考了 Grok 给出的算法框架建议（局部极值点 + 水平聚类 + 强度打分），采纳了其中 ATR prominence 过滤和动态容忍度的思路。

### 算法优化点

#### 1. ATR Prominence 过滤（新增）
- 新增 `calcATR()` 方法，计算 14 周期 ATR
- `detectSwingPoints()` 增加 ATR 显著性过滤：swing 的幅度必须 >= ATR × 0.5
- 过滤掉微小波动产生的噪音 swing point
- 效果：ETH 1h 的 13 个 swing point 全部通过（ATR=18.86, 最小幅度 21.03），说明过滤阈值合理

#### 2. ATR 动态聚类容忍度（改进）
- 聚类容忍度从固定 0.3% 改为 `ATR% × 0.5`
- 范围限制：最小 0.2%，最大 1.0%
- 效果：高波动品种（BTC）自动放宽容忍度，低波动品种（ADA）自动收紧

#### 3. 整数关口步长过滤（改进）
- `detectRoundNumbers()` 增加 `minStep` 过滤：步长必须 >= 价格的 0.5%
- ETH 2350：step=10 被过滤（0.43%），step=50 保留（2.13%）
- 效果：消除了 ETH 的 2330/2340 等无意义的 10 倍数候选

#### 4. 评分系统优化（改进）
- 触及次数评分增加高频档位：
  - ≥10 次: 4 分（新增）
  - ≥4 次: 3 分
  - ≥3 次: 2 分
  - ≥2 次: 1 分
- 效果：ETH 2393.33（18 次触及）从 mid 提升为 strong

#### 5. 最小触碰次数（改进）
- 从 `touchCount >= 1` 提高到 `touchCount >= 2`
- 整数关口不再单独作为入选条件（不再有无触及的纯整数关口）
- 密集成交区（volume cluster）仍可独立入选

#### 6. 选位逻辑简化（改进）
- 去掉"每种强度保证选一个"的强制分配
- 纯按分数排序取前 3 个，质量优先

#### 7. ATR 动态搜索范围（新增 2026-04-15）
- 距离过滤从固定 ±3% 改为 `±max(3%, ATR%×3)`，上限 8%
- 高波动币种（如 ZEC，ATR=1.85%）搜索范围自动扩大到 ±5.55%
- 低波动币种保持 ±3% 不变
- 效果：ZEC 能捕获到 372（+3.5%）等更远的真实阻力位

#### 8. 合并容忍度提升（改进 2026-04-15）
- 从 `ATR% × 0.5` 提升到 `ATR% × 1.0`（一个 ATR 范围内视为同一区域）
- 范围限制：最小 0.3%，最大 2.0%
- 同方向的相近价位合并为一个，取加权结果
- 效果：ZEC 的 364 和 370 合并为 364.03，避免同方向太近的重复价位

## 算法流程（优化后）

```
detectKeyLevels(klines)
    │
    ├─ 1. calcATR(klines, 14) → ATR 绝对值 + 百分比
    │
    ├─ 2. detectSwingPoints(klines, atr)
    │      └─ ATR prominence 过滤：swing 幅度 >= ATR × 0.5
    │
    ├─ 3. detectVolumeClusters(klines) → 50 buckets, top 20%
    │
    ├─ 4. detectRoundNumbers(currentPrice)
    │      └─ minStep = price × 0.5% 过滤无意义步长
    │
    ├─ 5. 方向重分类：以当前价为基准
    │
    ├─ 6. 距离过滤：±3%（fallback ±5%）
    │
    ├─ 7. 触碰计数：仅统计近 60 根 K 线
    │
    ├─ 8. 过滤：touchCount >= 2 或 isVolumeCluster
    │
    ├─ 9. mergeAndClassify(candidates, type, price, tolerance)
    │      ├─ 聚类：tolerance = ATR% × 0.5 (min 0.2%, max 1.0%)
    │      ├─ 评分：touch(0-4) + volume(0-2) + round(0-1)
    │      └─ 选位：按分数取前 3，纯质量排序
    │
    └─ 10. Upsert → key_levels_v2
```
